Performance Engineer Agent

Visual map of your end-to-end performance lifecycle agent for JMeter scripting, enhancement, load execution, and reporting.

Agent Scope

  • Plan workload model and lifecycle checklist.
  • Generate base JMX and enhance existing JMX.
  • Build scenario-driven JMX from YAML or JSON.
  • Run local JMeter tests in non-GUI mode.
  • Produce Markdown or JSON SLA reports from JTL.

Available Commands

  • create-plan
  • scenario-template
  • create-scenario
  • generate-jmx / enhance-jmx
  • run-local
  • report

Lifecycle Flow

Step 1

Plan

Define users, ramp, duration, test types, and SLA targets in one lifecycle plan file.

Step 2

Template

Create starter scenario YAML with defaults, CSV config, and transaction journey.

Step 3

Script

Generate multi-transaction JMX with thread groups, samplers, headers, assertions, and extractors.

Step 4

Execute

Run JMeter non-GUI load tests with runtime properties and collect JTL output.

Step 5

Report

Compute p90, p95, p99, error rate, throughput, and SLA pass/fail by endpoint.

1) Plan + Scenario Scaffold

python -m perf_agent.agent create-plan --application "PetStore" --out performance_test_plan.md
python -m perf_agent.agent scenario-template --out scenario.yaml --application "PetStore" --domain "petstore.octoperf.com"

2) Build JMX from Scenario

python -m perf_agent.agent create-scenario --scenario scenario.yaml --out petstore_scenario.jmx

3) Run Load Test

python -m perf_agent.agent run-local --jmx petstore_scenario.jmx --jtl petstore_results.jtl

4) Generate SLA Report

python -m perf_agent.agent report --jtl petstore_results.jtl --out perf_report.md --sla-p95-ms 800 --sla-error-rate 1

Scenario YAML Shape

test_name: "PetStore Scenario"
defaults:
  domain: "petstore.octoperf.com"
  protocol: "https"
  headers:
    Accept: "application/json"
  threads: 50
  ramp: 120
  duration: 1800
csv:
  path: "data/users.csv"
  vars: "user,pass"
timer_ms: 500
thread_groups:
  - name: "Primary Users"
    threads: 50
    ramp: 120
    duration: 1800
    transactions:
      - name: "Launch"
        method: "GET"
        path: "/"
      - name: "Login"
        method: "POST"
        path: "/api/login"
        params:
          username: "${user}"
          password: "${pass}"
        assert:
          field: "ResponseCode"
          pattern: "200"
        extractor:
          ref: "sessionToken"
          regex: "\"token\":\"(.*?)\""
        think_time_ms: 1000
result_jtl: "results.jtl"
